Так, це варто додати. І з точки зору UX це навіть не “додаткова кнопка”, а обов’язковий елемент режимів типу:

* створення зв’язку
* створення anchor на edge
* будь-який modal/tool mode, де користувач “застрягає” до завершення дії

Бо якщо користувач увійшов у режим `add-connector` або `add-edge-anchor`, він повинен мати **явний і швидкий вихід**.

---

# Що найкраще з точки зору UX

Я б радив не просто кнопку “відмінити”, а **панель активного режиму** в секції connectors/tools.

Тобто коли режим не активний — показується звичайний набір кнопок.
Коли режим активний — зверху або в самій секції з’являється невеликий статус-блок:

### приклад

* **Режим:** Створення зв’язку
* якщо вже вибрано першу точку:

  * **Початок вибрано**
* кнопка:

  * **Скасувати**

Аналогічно для anchor:

* **Режим:** Додавання точки прив’язки
* **Клікніть по лінії**
* **Скасувати**

Це набагато краще, ніж просто маленька іконка без контексту.

---

# Найкращий патерн

## 1. Показувати активний стан інструмента

Кнопка “connection” або “add anchor” має бути visibly active.

## 2. Показувати окрему кнопку скасування тільки коли режим активний

Не тримати її постійно.

## 3. Дати кілька способів виходу

Користувач повинен мати змогу скасувати через:

* кнопку в панелі
* `Esc`
* клік по пустому canvas, якщо це не конфліктує з логікою редактора

Оце вже хороший UX.

---

# Як би я це зробив у твоєму редакторі

У тебе вже є `mode` і `tempSource`. Це дуже добре, бо cancel логіка вже майже готова.

---

## Які стани треба підтримати

### Для створення зв’язку

* `mode === "add-connector"`
* `tempSource === null` — ще не вибрано першу точку
* `tempSource !== null` — уже вибрано початок зв’язку

### Для створення anchor

* `mode === "add-edge-anchor"`

---

# Яку поведінку раджу

---

## Коли активний `add-connector`

У панелі показати:

### якщо `tempSource == null`

* текст: `Оберіть початкову точку зв’язку`
* кнопка: `Скасувати`

### якщо `tempSource != null`

* текст: `Оберіть кінцеву точку зв’язку`
* кнопка: `Скасувати`

Це дуже важливо, бо користувач бачить не просто “режим активний”, а що саме зараз треба зробити.

---

## Коли активний `add-edge-anchor`

У панелі показати:

* текст: `Клікніть по лінії, щоб додати точку прив’язки`
* кнопка: `Скасувати`

---

# Що має робити “Скасувати”

Це має бути одна централізована action у controller або store.

Наприклад:

```js
cancelActiveTool()
```

Логіка така:

```js id="oo5mf4"
cancelActiveTool() {
  const store = useCanvasStore.getState();

  store.setMode("select");
  store.setTempSource(null);
  store.setHoveredEdgeAnchorId(null);
  store.clearContextMenu?.();
}
```

Можна ще скидати preview, якщо він буде пізніше.

---

# Найкраща структура для UI

Я б зробив у панелі connectors такий блок:

---

## Коли режим не активний

Показуєш звичайні кнопки:

* Створити зв’язок
* Додати anchor

---

## Коли режим активний

Показуєш info-bar / action-bar:

### для add-connector

```text
Створення зв’язку
[Оберіть початкову точку]
[Скасувати]
```

або

```text
Створення зв’язку
[Оберіть кінцеву точку]
[Скасувати]
```

### для add-edge-anchor

```text
Додавання точки прив’язки
[Клікніть по лінії]
[Скасувати]
```

---

# Чому так краще, ніж просто іконка “х”

Бо одна маленька іконка без тексту:

* менш помітна
* не пояснює, що відбувається
* гірше для нових користувачів

А невеликий режимний блок:

* показує контекст
* показує наступну дію
* дає явний вихід

Це значно кращий UX.

---

# Якщо хочеш мінімалістично

Якщо панель вузька і не хочеш великий блок, можна зробити так:

### активний tool button + сусідня кнопка cancel

Наприклад:

* кнопка link активна
* поруч з’являється кнопка з іконкою `Close` / `Cancel`

Але навіть тоді я б додав tooltip:

* `Скасувати створення зв’язку`
* `Скасувати додавання точки прив’язки`

---

# Який UX я раджу як best practice

Ось оптимальний набір:

## 1. Sticky active mode indicator

В панелі видно, що зараз активний режим.

## 2. Secondary cancel action

Поруч є явне `Скасувати`.

## 3. `Esc` для power users

При натисканні `Esc` режим скидається.

## 4. Автоматичний вихід після успішного створення

Після створення edge або anchor:

* або повертати в `select`
* або лишати режим активним, якщо користувач малює серію зв’язків

Для старту я б радив:

* після anchor creation → `select`
* після connection creation → залежить від сценарію, але теж можна в `select`

---

# Що краще: автоматично виходити чи лишатися в режимі

Це важливе UX-рішення.

---

## Для зв’язків

Є 2 варіанти:

### Варіант A — auto exit

Після створення одного зв’язку повертаєш у `select`.

**Плюси:**

* менше шансів “застрягти” в режимі
* краще для більшості користувачів

**Мінуси:**

* незручно, якщо треба намалювати багато зв’язків підряд

### Варіант B — stay in mode

Після створення одного edge режим лишається активним.

**Плюси:**

* швидше для інтенсивного редагування

**Мінуси:**

* користувач може забути, що режим ще активний

### Моя порада

Для початку:

* після створення **anchor** — виходити в `select`
* після створення **edge** — теж виходити в `select`

А потім, якщо захочеш pro-mode, додаси “липкий режим”.

# Як це реалізувати в store/controller

---

## Store вже має майже все потрібне

Тобі достатньо:

* `mode`
* `tempSource`

Можна ще додати helper state:

```js id="waur8b"
isToolActive: mode === "add-connector" || mode === "add-edge-anchor"
```

але це краще просто в компоненті обчислювати.

---

## Controller

Додай метод:

```js id="fiuaot"
cancelActiveTool() {
  const store = useCanvasStore.getState();

  store.setMode("select");
  store.setTempSource(null);
  store.setHoveredEdgeAnchorId(null);
}
```

Якщо буде preview line:

```js
store.setConnectionPreview(null);
```

---

# Як підключити `Esc`

У canvas або editor wrapper:

```js id="83mp3h"
useEffect(() => {
  const onKeyDown = (e) => {
    if (e.key === "Escape") {
      canvasController.cancelActiveTool();
    }
  };

  window.addEventListener("keydown", onKeyDown);
  return () => window.removeEventListener("keydown", onKeyDown);
}, []);
```

Це дуже корисно.

---

# Як це виглядатиме в панелі

Приклад логіки в компоненті панелі:

```js id="3sg8p0"
const mode = useCanvasStore((s) => s.mode);
const tempSource = useCanvasStore((s) => s.tempSource);

const isAddConnector = mode === "add-connector";
const isAddEdgeAnchor = mode === "add-edge-anchor";
```

## Рендер

```jsx id="dfe4z2"
{isAddConnector && (
  <Box>
    <Typography variant="body2" fontWeight={600}>

      Клікніть по лінії
    </Typography>

    <Button
      size="small"
      color="inherit"

      Скасувати
```

# Найкраща деталь UX

Я б ще радив при активному режимі трохи підсвічувати canvas state:

### для add-connector

* показувати hover на anchors
* змінювати cursor
* текст у панелі

### для add-edge-anchor

* підсвічувати edges, до яких можна додати anchor

Тоді `Скасувати` буде частиною цілісного сценарію, а не випадковою кнопкою.

# Моя практична рекомендація

Найкращий варіант для тебе зараз:

## У панелі connectors:

* залишити кнопки інструментів
* коли режим активний — показувати маленький info/action блок
* у цьому блоці:

  * назва режиму
  * коротка інструкція
  * кнопка `Скасувати`

## Додатково:

* підтримати `Esc`
* після успішного завершення повертати в `select`

Це найчистіший і найзручніший UX.
