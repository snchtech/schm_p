Так, причина в тебе вже майже знайдена.

## Чому зараз так відбувається

У твоєму поточному MVP `moveElbowSegmentHandle()` робить це:

```js id="1r410r"
if (!waypoints.length) {
  waypoints.push({
    id: crypto.randomUUID(),
    x: handle.x,
    y: handle.y,
  });
}
```

Тобто **будь-який drag синього квадратного handle** автоматично створює `waypoint`.
Через це початковий segment-handle поводиться як “Додати точку шляху”.

А ти хочеш інше:

* **початкові сині квадратики** = тимчасові **segment handles**

  * вони не створюють `waypoint`
  * вони просто зміщують базовий сегмент
* **точки, додані через “Додати точку шляху”** = справжні **waypoints**

  * саме вони створюють додаткове кероване коліно
  * і саме їх можна видаляти як path point

* **segment handle drag** може змінити форму polyline

# 2. Онови `buildOrthogonalElbowPoints()` у `connectorGeometry.js`

Ось головна зміна.

```js id="zjffmf"
export function buildOrthogonalElbowPoints(start, end, route = {}) {
  const waypoints = route?.waypoints || [];
  const baseOffset = route?.baseOffset || null;

  // ===== 1. якщо є user waypoints -> будуємо через них
  if (waypoints.length) {
    const points = [start.x, start.y];
    let current = { ...start };

    for (const wp of waypoints) {
      if (!same(current.x, wp.x)) {
        points.push(wp.x, current.y);
        current = { x: wp.x, y: current.y };
      }

      if (!same(current.y, wp.y)) {
        points.push(current.x, wp.y);
        current = { x: current.x, y: wp.y };
      }
    }

    if (!same(current.x, end.x)) {
      points.push(end.x, current.y);
      current = { x: end.x, y: current.y };
    }

    if (!same(current.y, end.y)) {
      points.push(current.x, end.y);
    }

    return normalizePoints(points);
  }

  // ===== 2. якщо waypoint ще нема, але є baseOffset
  // це НЕ path point, це лише зміщення базового сегмента
  if (baseOffset?.axis === "y") {
    return normalizePoints([
      start.x, start.y,
      start.x, baseOffset.value,
      end.x, baseOffset.value,
      end.x, end.y,
    ]);
  }

  if (baseOffset?.axis === "x") {
    return normalizePoints([
      start.x, start.y,
      baseOffset.value, start.y,
      baseOffset.value, end.y,
      end.x, end.y,
    ]);
  }

  // ===== 3. дефолтний elbow
  return normalizePoints([
    start.x, start.y,
    end.x, start.y,
    end.x, end.y,
  ]);
}
```

# 3. Перепиши `moveElbowSegmentHandle()` у `connectorEditing.js`

Ось найважливіша зміна:
**segment handle більше не створює waypoint.**

```js id="4aq7ct"
export function moveElbowSegmentHandle(connector, handle, pos) {
  const route = {
    ...(connector.route || {}),
  };

  // Якщо вже є user waypoints — ці ручки більше не керують базовим сегментом
  // Для MVP просто не чіпаємо їх тут.
  if ((route.waypoints || []).length > 0) {
    return connector;
  }

  if (handle.orientation === "horizontal") {
    route.baseOffset = {
      axis: "y",
      value: pos.y,
    };
  } else if (handle.orientation === "vertical") {
    route.baseOffset = {
      axis: "x",
      value: pos.x,
    };
  }

  return {
    ...connector,
    route,
  };
}
```

export function createWaypointAtSegment(connector, segmentIndex) {
  const points = connector.geometry?.points || [];
  const segments = getConnectorSegments(points);
  const seg = segments.find((s) => s.index === segmentIndex);

  if (!seg) return null;

  return {
    id: crypto.randomUUID(),
    x: seg.cx,
    y: seg.cy,
    axis: seg.orientation === "horizontal" ? "y" : "x",

# 7. Що це дасть по факту

Після цих змін поведінка стане такою:

## стартовий elbow connector

* має базові сині квадратики по центру сегментів
* drag цих квадратиків **не створює path point**
* він лише змінює `route.baseOffset`
* форма лінії змінюється, але це ще не `waypoint`

## після “Додати точку шляху”

* у `route.waypoints[]` з’являється справжня точка
* вона стає жовтою draggable точкою
* її можна видалити через context menu
* рухається тільки по потрібній осі

Це саме той поділ, який ти хотів.
