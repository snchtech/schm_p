Добре. Це **третій і дуже важливий крок архітектури** — зробити **єдиний registry елементів**.
Це прибере дублювання між:

* `ELEMENT_REGISTRY`
* `GraphLibrary`
* `ELEMENT_COMPONENTS`

і зробить систему значно простішою для масштабування.

Це саме той підхід, який використовують редактори типу **Figma / Miro / tldraw**.

---

# 1️⃣ Проблема поточної архітектури

Зараз у тебе приблизно так:

```
registry
   ↓
defaults

GraphLibrary
   ↓
shape components

DiagramSymbols
   ↓
ELEMENT_COMPONENTS mapping
```

Тобто один і той самий тип елемента описаний **у трьох місцях**.

Наприклад `twoRectangle`:

```
registry
GraphLibrary
DiagramSymbols
```

Це дуже швидко призводить до хаосу.

---

# 2️⃣ Правильна архітектура

Registry має містити **все про елемент**:

```
ELEMENT_REGISTRY
   │
   ├ component
   ├ defaults
   ├ anchors
   ├ preview
   └ metadata
```

Тоді **будь-який код бере все з одного місця**.

---

# 3️⃣ Нова структура папок (рекомендована)

```
graph
   registry.js
   shapes
        RectShape.jsx
        CircleShape.jsx
        TwoRectangleShape.jsx
```

# 5️⃣ Створи новий registry

### `graph/registry.js`

```javascript
import RectShape from "./shapes/RectShape";
import CircleShape from "./shapes/CircleShape";
import TwoRectangleShape from "./shapes/TwoRectangleShape";

export const ELEMENT_REGISTRY = {

  rect: {
    title: "Rectangle",

    component: RectShape,

    defaults: {
      w: 120,
      h: 60
    }
  },

  circle: {
    title: "Circle",

    component: CircleShape,

    defaults: {
      r: 40
    }
  },

  twoRectangle: {
    title: "Two rectangles",

    component: TwoRectangleShape,

    defaults: {
      w: 120,
      h: 50
    }
  }

};
```

# 6️⃣ Спрощуємо DiagramSymbolItem

Тепер **не потрібен `ELEMENT_COMPONENTS`**.

Відкрий:

```
DiagramSymbolItem.jsx
```

і заміни:

```javascript
const Shape = ELEMENT_COMPONENTS[el.type]
```

на:

```javascript
import { ELEMENT_REGISTRY } from "../graph/registry";

const Shape = ELEMENT_REGISTRY[el.type]?.component;
```

---

### Повний фрагмент

```javascript
const Shape = ELEMENT_REGISTRY[el.type]?.component;
```

---

### Рендер

```jsx
{Shape && <Shape el={el} />}
```

# 7️⃣ createElementFromTool тепер ще простіший

### `createElementFromTool.js`

```javascript
import { ELEMENT_REGISTRY } from "../graph/registry";

export function createElementFromTool(type, x, y) {

  const config = ELEMENT_REGISTRY[type];
  if (!config) return null;

  return {
    id: crypto.randomUUID(),
    type,
    x,
    y,
    ...config.defaults
  };

}
```

# 11️⃣ Це ще не все — можна зробити ще краще

Наступний крок (дуже важливий):

### додати anchors у registry

Наприклад:

```javascript
twoRectangle: {
  component: TwoRectangleShape,

  defaults: { w:120, h:50 },

  anchors: (el) => [
    {x:0, y:el.h/2},
    {x:el.w, y:el.h/2},
    {x:el.w/2, y:0},
    {x:el.w/2, y:el.h*2}
  ]
}
```

І тоді:

```
getAnchorPoints
```

взагалі не потрібна.
