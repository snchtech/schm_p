Добре. Тепер зробимо **один з найсильніших архітектурних кроків — винесемо Graph State у Zustand store**.
Це те, що реально використовують більшість canvas-редакторів (включно з tldraw).

Це дасть тобі:

✔ менше props
✔ централізований graph state
✔ прості undo/redo
✔ кращу продуктивність
✔ простіший код

Я покажу **покроково**, щоб не зламати твій редактор.

## Код store

```javascript
import { create } from "zustand";

export const useGraphStore = create((set, get) => ({

  nodes: [],
  edges: [],

  selection: null,

  hoveredNode: null,
  hoveredAnchor: null,

  setNodes: (fn) =>
    set((state) => ({
      nodes: typeof fn === "function" ? fn(state.nodes) : fn
    })),

  addNode: (node) =>
    set((state) => ({
      nodes: [...state.nodes, node]
    })),

  updateNodePosition: (id, x, y) =>
    set((state) => ({
      nodes: state.nodes.map((n) =>
        n.id === id ? { ...n, x, y } : n
      )
    })),

  deleteNode: (id) =>
    set((state) => ({
      nodes: state.nodes.filter((n) => n.id !== id),
      edges: state.edges.filter(
        (e) => e.from !== id && e.to !== id
      )
    })),

  setSelection: (selection) => set({ selection }),

  setHoveredNode: (id) => set({ hoveredNode: id }),

  setHoveredAnchor: (anchor) => set({ hoveredAnchor: anchor }),

}));
```

# Крок 4. Оновити Stage handler

Було:

```javascript
S.setSymbols((prev) => [...prev, newElement])
```

Тепер:

```javascript
addNode(newElement)
```

---

### Повна функція

```javascript
const handleStageMouseDown = (e) => {

  const stage = e.target.getStage()
  if (!stage) return

  if (e.target !== stage) return

  const pointer = stage.getPointerPosition()
  if (!pointer) return

  if (editorMode === "add-node" && activeTool) {

    const newElement = createElementFromTool(
      activeTool,
      pointer.x,
      pointer.y
    )

    if (!newElement) return

    addNode(newElement)

  }

}
```

# 🚀 Наступний крок (дуже важливий)

Тепер, коли store з'явився, можна зробити **одну з найсильніших оптимізацій редактора**:

### CanvasController

Він буде керувати:

```
mouseDown
mouseMove
mouseUp
drag
selection
creation
connection
```

І Stage буде мати **один handler замість десятків**.

Це зменшує складність редактора **в 2-3 рази**.

Якщо хочеш — покажу як це зробити.
