Так. Розкладемо це по файлах, щоб у тебе вийшла **робоча схема додавання елементів на основний Konva Stage після кліку по SVG-іконці в панелі**.

Я покажу базовий варіант:

* у `ToolPalette` вибирається тип елемента
* цей тип передається в `DiagramEditor`
* при кліку по порожньому місцю `Stage` створюється новий елемент
* `DiagramSymbols` його відмалює через `Group`

---

# 1. Що має вийти логічно

Потік буде такий:

```text
ToolPalette
  -> onSelectTool("twoRectangle")

activeTool у батьківському компоненті
  -> передається в DiagramEditor

DiagramEditor
  -> при кліку по Stage створює новий object
  -> додає його в S.symbols

DiagramSymbols
  -> проходить symbols.map(...)
  -> для type="twoRectangle" рендерить потрібний shape
```

# 4. Створи helper `createElementFromTool.js`

Це не обов’язково, але дуже рекомендую.
Наприклад файл:

`/components/diagram/createElementFromTool.js`

```javascript
import { ELEMENT_REGISTRY } from "../graph/registry";

export function createElementFromTool(type, x, y) {
  const config = ELEMENT_REGISTRY[type];

  if (!config) return null;

  return {
    id: crypto.randomUUID(),
    type,
    x,
    y,
    ...config.defaults,
  };
}
```

## 7.3 Додай функцію кліку по Stage

Всередині `DiagramEditor` додай:

```javascript
const handleStageMouseDown = (e) => {
  // якщо tool не вибраний — нічого не робимо
  if (!activeTool) return;

  // створюємо елемент лише при кліку по порожньому canvas,
  // а не по вже існуючому shape
  const stage = e.target.getStage();
  if (e.target !== stage) return;

  const pointer = stage.getPointerPosition();
  if (!pointer) return;

  const newElement = createElementFromTool(activeTool, pointer.x, pointer.y);
  if (!newElement) return;

  S.setSymbols((prev) => [...prev, newElement]);
};
```

## Але в тебе зараз є серйозна проблема в `symbols.map`

У твоєму коді зараз всередині `map` стоїть `const Shape = ...` і `useEffect(...)` прямо всередині callback `map`, а так не можна.

**Хуки не можна викликати всередині циклів, умов чи `map`.**

Тут треба винести один елемент в окремий компонент.

# 10. Як правильно переробити `DiagramSymbols.jsx`

Ось правильна структура.

```javascript
"use client";

import React, { useEffect, useRef } from "react";
import { Group, Rect, Circle } from "react-konva";
import GraphLibrary from "../graph";

const ELEMENT_COMPONENTS = {
  rect: GraphLibrary.rect,
  circle: GraphLibrary.circle,
  twoRectangle: GraphLibrary.twoRectangle,
  line: GraphLibrary.line,
  text: GraphLibrary.text,
};

function DiagramSymbolItem({
  el,
  isAddingConnector,
  hoveredElement,
  hoveredAnchor,
  getAnchorPoints,
  setHoveredElement,
  setHoveredAnchor,
  handleDragMove,
  handleAnchorClick,
  onElementContextMenu,
  onMouseDown,
  isSelected,
  registerNodeRef,
}) {
  const groupRef = useRef(null);
  const Shape = ELEMENT_COMPONENTS[el.type];

  useEffect(() => {
    if (!registerNodeRef) return;

    registerNodeRef(el.id, groupRef.current);

    return () => {
      registerNodeRef(el.id, null);
    };
  }, [el.id, registerNodeRef]);

  return (
    <Group
      ref={groupRef}
      x={el.x}
      y={el.y}
      draggable
      onDragMove={(e) => handleDragMove(el.id, e)}
      onMouseEnter={() => isAddingConnector && setHoveredElement(el.id)}
      onMouseLeave={() => isAddingConnector && setHoveredElement(null)}
      onContextMenu={(e) => onElementContextMenu(e, { type: "node", id: el.id })}
      onMouseDown={() => onMouseDown(el)}
    >
      {Shape && <Shape el={el} />}

      {isSelected && el.w && el.h && (
        <Rect
          x={-6}
          y={-6}
          width={el.w + 12}
          height={el.h + 12}
          cornerRadius={12}
          stroke="#3b82f6"
          strokeWidth={1}
          dash={[6, 4]}
          listening={false}
        />
      )}

      {hoveredElement === el.id &&
        getAnchorPoints(el).map((point, i) => (
          <Circle
            key={i}
            x={point.x - el.x}
            y={point.y - el.y}
            radius={6}
            fill={
              hoveredAnchor &&
              hoveredAnchor.x === point.x &&
              hoveredAnchor.y === point.y
                ? "yellow"
                : "green"
            }
            stroke="black"
            strokeWidth={1}
            onMouseEnter={() => setHoveredAnchor(point)}
            onMouseLeave={() => setHoveredAnchor(null)}
            onClick={() =>
              handleAnchorClick({
                ...point,
                parentId: el.id,
                kind: "symbolAnchor",
              })
            }
          />
        ))}
    </Group>
  );
}

export const DiagramSymbols = ({
  symbols,
  isAddingConnector,
  hoveredElement,
  hoveredAnchor,
  getAnchorPoints,
  setHoveredElement,
  setHoveredAnchor,
  handleDragMove,
  handleAnchorClick,
  onElementContextMenu,
  onMouseDown,
  isSelected,
  registerNodeRef,
}) => {
  return (
    <>
      {symbols.map((el) => (
        <DiagramSymbolItem
          key={el.id}
          el={el}
          isAddingConnector={isAddingConnector}
          hoveredElement={hoveredElement}
          hoveredAnchor={hoveredAnchor}
          getAnchorPoints={getAnchorPoints}
          setHoveredElement={setHoveredElement}
          setHoveredAnchor={setHoveredAnchor}
          handleDragMove={handleDragMove}
          handleAnchorClick={handleAnchorClick}
          onElementContextMenu={onElementContextMenu}
          onMouseDown={onMouseDown}
          isSelected={isSelected?.id === el.id}
          registerNodeRef={registerNodeRef}
        />
      ))}
    </>
  );
};
```

# 12. Важливий нюанс: чи треба скидати tool після вставки

Є два режими.

## Варіант 1 — одна вставка

Після додавання елемента tool очищається:

```javascript
onSelectTool(null)
```

Тоді кожен елемент вставляється по окремому вибору інструмента.

## Варіант 2 — багато вставок підряд

Tool залишається активним, і можна багато разів клацати по сцені.

На старті я б радив **другий варіант**. Він простіший і зручний.

# 15. На що звернути увагу у твоєму поточному `DiagramEditor`

У тебе в коді є кілька друкарських помилок, які завадять роботі. Наприклад:

* `ukeRef(null)` замість `useRef(null)`
* `nul1` замість `null`
* `clientx` замість `clientX`
* `Iselected` замість `!selected` або `if (!selected) return`
* `apiRef.current ?. destroy()` — пробіл між `current` і `?.` краще прибрати
* імпорт `GraphContextMenuMui` має дивний шлях `./ .. /graph/interface-elements/...`
* `SelectionTransformer` імпортується як `selectionTransformer`, а використовується як компонент `SelectionTransformer`

Тобто перед інтеграцією логіки додавання елементів краще ще трохи почистити синтаксис.

# 16. Найпростіший робочий варіант `DiagramEditor.jsx`

Ось компактний приклад саме під додавання елементів:

```javascript
"use client";

import React, { useEffect, useRef, useState } from "react";
import { Stage, Layer } from "react-konva";
import { useDiagramState } from "./useDiagramState";
import { DiagramConnections } from "./DiagramConnections";
import { DiagramSymbols } from "./DiagramSymbols";
import { setupLayerZoomPan } from "./LayerZoom";
import { createElementFromTool } from "./createElementFromTool";

export default function DiagramEditor({ activeTool }) {
  const stageRef = useRef(null);
  const layerRef = useRef(null);
  const apiRef = useRef(null);
  const nodeRefs = useRef(new Map());

  const registerNodeRef = (id, node) => {
    if (!node) nodeRefs.current.delete(id);
    else nodeRefs.current.set(id, node);
  };

  const S = useDiagramState();

  useEffect(() => {
    const stage = stageRef.current;
    const layer = layerRef.current;
    if (!stage || !layer) return;

    apiRef.current = setupLayerZoomPan(stage, layer, { panMode: "mmb_or_space" });

    return () => {
      apiRef.current?.destroy();
    };
  }, []);

  const handleStageMouseDown = (e) => {
    if (!activeTool) return;

    const stage = e.target.getStage();
    if (e.target !== stage) return;

    const pointer = stage.getPointerPosition();
    if (!pointer) return;

    const newElement = createElementFromTool(activeTool, pointer.x, pointer.y);
    if (!newElement) return;

    S.setSymbols((prev) => [...prev, newElement]);
  };

  return (
    <section style={{ border: "1px solid #ccc", display: "inline-block" }}>
      <Stage
        ref={stageRef}
        width={1000}
        height={600}
        style={{ border: "1px solid #ccc" }}
        onMouseDown={handleStageMouseDown}
      >
        <Layer ref={layerRef}>
          <DiagramConnections
            connections={S.connections}
            hoveredConnectionId={S.hoveredConnectionId}
            hoveredLineAnchorId={S.hoveredLineAnchorId}
            isAddingConnector={S.isAddingConnector}
            setHoveredConnectionId={S.setHoveredConnectionId}
            setHoveredLineAnchorId={S.setHoveredLineAnchorId}
            handleLineClick={S.handleLineClick}
            handleAnchorClick={S.handleAnchorClick}
            updateConnectionOffset={S.updateConnectionOffset}
          />

          <DiagramSymbols
            symbols={S.symbols}
            isAddingConnector={S.isAddingConnector}
            hoveredElement={S.hoveredElement}
            hoveredAnchor={S.hoveredAnchor}
            getAnchorPoints={S.getAnchorPoints}
            setHoveredElement={S.setHoveredElement}
            setHoveredAnchor={S.setHoveredAnchor}
            handleDragMove={S.handleDragMove}
            handleAnchorClick={S.handleAnchorClick}
            onElementContextMenu={() => {}}
            onMouseDown={S.selectHightlightEl}
            isSelected={S.selected}
            registerNodeRef={registerNodeRef}
          />
        </Layer>
      </Stage>
    </section>
  );
}
```

# 17. Що я б радив тобі зробити далі

Найкращий порядок такий:

1. спочатку зробити **вставку елемента по кліку на Stage**
2. потім перевірити, що `twoRectangle` рендериться
3. потім уже доробити:

   * selection
   * transformer
   * context menu
   * connectors

Бо якщо спочатку стабільно не працює базова вставка shape, далі редактор буде важко дебажити.

Якщо хочеш, я в наступному повідомленні можу вже зібрати тобі **повністю готовий комплект із 5 файлів** у фінальному вигляді, щоб ти просто вставив їх у проект.
