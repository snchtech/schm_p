Ти зараз описав вже не просто "лінію", а механіку типу:

```txt
smart alignment / smart guides / snapping system
```

І це абсолютно правильний напрямок.

І головне:
це треба реалізовувати НЕ в connector layer, а як окрему систему canvas snapping/alignment.

Бо:

* це буде працювати не тільки для ліній
* а для всіх node
* і для layout
* і для move
* і для resize
* і для connectors

---

# 1. Головна ідея

Твоя проблема:

```txt
free-line
```

не може автоматично стати:

* строго горизонтальною
* строго вертикальною

бо:

* фігури не вирівняні

І це нормально.

Тому правильне рішення:
НЕ constrain line.

А:

# align nodes.

---

# 2. Як це працює в draw.io / Figma

Коли рухаєш node:

система:

* перевіряє alignment
* показує guide lines
* робить snapping

І тоді:

* центри збігаються
* edges збігаються
* connector автоматично стає рівним

# 11. Алгоритм

При drag node:

---

## КРОК 1

Беремо bounds node:

```js
movingBox
```

---

## КРОК 2

Перебираємо всі інші nodes:

```js
otherBoxes
```

---

## КРОК 3

Шукаємо:

```txt
abs(centerX1 - centerX2) < SNAP_THRESHOLD
```

---

## КРОК 4

Якщо знайдено:

* додаємо guide
* робимо snap

# 15. Чому це вирішує проблему connector-а

Бо connector already attached.

У тебе вже є:

```js
attached: {
   type: "node"
}
```



і:

```js
rebuildConnectorGeometry()
```

Тому:

* node вирівнявся
* connector автоматично перебудувався
* line стала рівною
