Для реалізації цієї логіки на рівні **Oracle 10g** (де вже є повна підтримка функцій для роботи з рядками та пакета `UTL_RAW`) найкраще створити окрему збережену функцію (Stored Function).

Оскільки версія 10g є досить старою, в ній вбудована функція `HEX2DEC` відсутня в стандартному SQL (вона з'явилася пізніше). Проте в Oracle 10g є пакет `UTL_RAW.CAST_TO_BINARY_INTEGER`, який ідеально підходить для конвертації реверсних байтів, або ж можна використати класичний математичний підхід чи пакет `TO_NUMBER` з маскою.

Нижче наведено два найкращих способи реалізації: перший — через чистий SQL (якщо код завжди строго 8 символів), другий — універсальна функція PL/SQL, яка сама відрізає зайве за вашою маскою `XXXXXXXX` та робить реверс.

---

### Спосіб 1: Чистий SQL-запит (для 8-значного HEX)

Якщо ваші вхідні дані — це рядки строго по 8 символів (наприклад, `12345678`), ви можете зробити реверс та конвертацію одним запитом за допомогою розбиття рядка на субрядки (`SUBSTR`):

```sql
SELECT TO_NUMBER(
         SUBSTR(hex_val, 7, 2) || 
         SUBSTR(hex_val, 5, 2) || 
         SUBSTR(hex_val, 3, 2) || 
         SUBSTR(hex_val, 1, 2), 
         'XXXXXXXX'
       ) as dec_val
FROM (SELECT '12345678' as hex_val FROM dual);

```

**Як це працює:**

1. `SUBSTR(hex_val, 7, 2)` бере 4-й байт (7 та 8 символи) і ставить на початок.
2. Перепутанні байти склеюються оператором `||` у новий HEX-рядок: `78563412`.
3. `TO_NUMBER(..., 'XXXXXXXX')` — це стандартна функція Oracle, яка приймає шістнадцятковий рядок і форматну маску (якраз вашу `'XXXXXXXX'`) та перетворює її на звичайне десяткове число.

---

### Спосіб 2: Універсальна PL/SQL функція (Рекомендовано)

Якщо коди у базі можуть бути різної довжини, або якщо ви хочете красиве рішення, яке можна викликати в будь-якому місці бази даних як `GET_DEC_NFC(hex_code)`, створіть таку функцію:

```sql
CREATE OR REPLACE FUNCTION get_dec_nfc (p_hex IN VARCHAR2) 
RETURN NUMBER 
IS
  v_clean_hex VARCHAR2(100);
  v_reversed  VARCHAR2(8);
  v_dec       NUMBER;
BEGIN
  -- 1. Очищаємо від можливих пробілів та переводимо у верхній регістр
  v_clean_hex := UPPER(TRIM(p_hex));
  
  -- 2. Робимо реверс перших 4-х байтів (8 символів)
  --    Імітуємо маску 'XXXXXXXX', беручи байти з кінця першої 8-символьної групи
  v_reversed := SUBSTR(v_clean_hex, 7, 2) || 
                SUBSTR(v_clean_hex, 5, 2) || 
                SUBSTR(v_clean_hex, 3, 2) || 
                SUBSTR(v_clean_hex, 1, 2);
                
  -- 3. Конвертуємо отриманий реверсний HEX у DEC за допомогою маски
  v_dec := TO_NUMBER(v_reversed, 'XXXXXXXX');
  
  RETURN v_dec;
EXCEPTION
  WHEN OTHERS THEN
    RETURN NULL; -- Повертає NULL, якщо на вхід прийшло сміття замість HEX
END;
/

```

#### Як цим користуватися в Oracle 10g:

Тепер ви можете дуже просто опрацьовувати ваші таблиці. Наприклад, якщо у вас є таблиця `nfc_tags` зі стовпчиком `hex_code`:

```sql
SELECT hex_code, get_dec_nfc(hex_code) as dec_code 
FROM nfc_tags;

```

**Результат роботи:**

* Якщо `hex_code` = `'12345678'`, функція поверне `2018915346`.
* Якщо `hex_code` має довжину більше ніж 8 символів (наприклад, 12 символів від міток Mifare 7-byte `12345678AABB`), завдяки логіці `SUBSTR` функція все одно візьме перші 4 байти, розгорне їх і застосує маску `XXXXXXXX`, видавши той самий правильний результат `2018915346`.
