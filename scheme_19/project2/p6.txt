```txt
----------------------------------------------------------------
-- Функція нормалізації NFC коду
----------------------------------------------------------------
FUNCTION NORMALIZE_NFC(
    P_NFC_CODE VARCHAR2
) RETURN VARCHAR2
IS

    V_CODE VARCHAR2(4000);

BEGIN

    ----------------------------------------------------------------
    -- NULL
    ----------------------------------------------------------------
    IF P_NFC_CODE IS NULL THEN
        RETURN NULL;
    END IF;

    ----------------------------------------------------------------
    -- Прибираємо пробіли та переводимо в upper
    ----------------------------------------------------------------
    V_CODE := UPPER(TRIM(P_NFC_CODE));

    ----------------------------------------------------------------
    -- Якщо вже число -> повертаємо як є
    ----------------------------------------------------------------
    IF REGEXP_LIKE(V_CODE, '^[0-9]+$') THEN
        RETURN V_CODE;
    END IF;

    ----------------------------------------------------------------
    -- Якщо HEX -> переводимо в DEC
    ----------------------------------------------------------------
    IF REGEXP_LIKE(V_CODE, '^[0-9A-F]+$') THEN

        RETURN TO_CHAR(
            TO_NUMBER(V_CODE, 'XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX')
        );

    END IF;

    ----------------------------------------------------------------
    -- Якщо формат невідомий -> повертаємо як є
    ----------------------------------------------------------------
    RETURN V_CODE;

EXCEPTION
    WHEN OTHERS THEN
        RETURN P_NFC_CODE;
END;
```

І тепер в основній процедурі змінюємо блок VIEW2:

Замість:

FOR R IN (
SELECT USER_ID,
NFC_CODE
FROM VIEW2
WHERE NFC_CODE IS NOT NULL
)

робимо:

FOR R IN (
SELECT USER_ID,
NORMALIZE_NFC(NFC_CODE) AS NFC_CODE
FROM VIEW2
WHERE NFC_CODE IS NOT NULL
)

Або ще краще прямо в PROCESS_NFC:

PROCESS_NFC(
P_USER_ID   => R.USER_ID,
P_NFC_CODE  => NORMALIZE_NFC(R.NFC_CODE),
P_SOURCE_ID => 2
);

## -- ВАЖЛИВИЙ МОМЕНТ

TO_NUMBER(..., 'XXXXXXXXXXXXXXXX')
працює максимум до 16 hex символів.

Тому я поставив:

'XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX'

щоб підтримувати довгі HEX значення.

Але Oracle NUMBER має обмеження ~38 цифр.

Якщо NFC може бути дуже довгим (20-30+ hex символів),
то краще:

* або зберігати як VARCHAR2
* або використовувати RAW

---

## -- Рекомендація

Краще зробити NFC_CODE VARCHAR2,
а не NUMBER.

Бо:

* можуть бути leading zeros
* можуть бути mixed formats
* можуть бути дуже довгі HEX
* менше проблем при інтеграціях
