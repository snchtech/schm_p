Так.
І це якраз НАЙПРАВИЛЬНІШЕ рішення для твоєї архітектури.

Тобто:

# тобі НЕ треба окремий moveBendVertex engine.

Бо після insertion bend:

* segments автоматично rebuild-яться
* segmentHandles автоматично rebuild-яться
* далі existing drag logic вже працює.

І це дуже близько до draw.io internal behavior.

---

# ПРАВИЛЬНА МОДЕЛЬ

Після:

```txt id="vq8wmd"
insert bend vertex
```

конектор стає:

# звичайним orthogonal connector.

---

# Тобто

ОСЬ ЦЕ:

```txt id="y0e4h6"
A -------- B
```

після add bend:

```txt id="y7pwmx"
A ----+
      |
      +----- B
```

це вже:

* 3 звичайні segments
* 1 звичайний bend vertex.

---

# І ДАЛІ

твій existing:

```txt id="xhz7dw"
move segment
```

має працювати без нової логіки.

---

# АЛЕ Є КРИТИЧНА УМОВА

Segment drag
повинен працювати:

# через vertices.

А НЕ:

```txt id="kg53l2"
через geometry.points
```

---

# ТЕПЕР ГОЛОВНА ІДЕЯ

Тобі треба:

# позначити vertex.type


* user-created elbow

# ВИСНОВОК

# moveBendVertex НЕ ПОТРІБЕН.

Правильний підхід:

```txt id="jlwm311"

   ↓

   ↓
existing segment drag logic
```

Але:

має працювати через vertices,
а НЕ geometry.points.

І тоді в тебе буде реально draw.io-like architecture.
