## 1️⃣ Розпізнаний код файлу (зі скріну)

**`witRoleAccess.js`** (є друкарська помилка в назві — має бути `withRoleAccess.js`, але я спочатку відтворюю як є):

```js
import { useAuth } from '@/context/AuthContext'

export function withRoleAccess(WrappedComponent, allowedRoles = []) {
  return function RoleProtectedComponent(props) {
    const { user } = useAuth()

    if (!user || !user.roles) {
      return <p>Access Denied</p>
    }

    const hasAccess = user.roles.some((role) => allowedRoles.includes(role))

    if (!hasAccess) {
      return <p>Access Denied</p>
    }

    return <WrappedComponent {...props} />
  }
}
```

## 2️⃣ Куди класти в новій структурі

Це **HOC-guard** (UI-рівень), який залежить від `AuthContext`, тому логічно класти в **shared/ui** поруч із `RoleProtectedComponent`.

✅ Пропоноване місце:

```
src/shared/ui/withRoleAccess.js
```

(або `src/shared/ui/guards/withRoleAccess.js`, якщо хочеш папку guards)
