Так, це вже наступний правильний рівень.

Зараз у тебе connector створюється як готовий об’єкт у фіксованих координатах.
А draw.io працює інакше:

* користувач **активує інструмент лінії**
* наводиться на фігуру
* бачить точки прив’язки
* тягне від однієї фігури до іншої
* під час drag бачить preview лінії
* відпускає на другій фігурі
* створюється connector, уже прив’язаний до двох об’єктів
* потім його можна:

  * від’єднати
  * переприв’язати
  * редагувати

Тобто тобі треба перейти від режиму:

**“кнопка вставила готову лінію в canvas”**

до режиму:

**“інструмент створення connector-а через drag-start → drag-preview → drop”**

---

# Що саме треба змінити концептуально

## Зараз

Ти створюєш connector приблизно так:

* натиснув кнопку
* створив `createConnector(...)`
* поклав у `edges`
* потім окремо його рухаєш/прив’язуєш

## Треба

Після натискання на кнопку не створювати одразу фінальний connector, а перейти в **режим побудови**.

Тобто кнопка має робити не:

```js
createTestConnector()
```

а щось типу:

```js
setMode("create-connector")
setPendingConnectorTemplate(...)
```

---

# Як це працює як у draw.io

Я б заклав таку послідовність.

## Етап 1. Вибір інструмента

Користувач обирає:

* ламану лінію
* пряму
* стрілку
* подвійну стрілку

У store зберігається шаблон майбутнього connector-а:

```js
{
  pathType: "orthogonal-elbow",
  endcaps: { start: "none", end: "none" },
  behavior: { type: "default", link: null }
}
```

---

## Етап 2. Hover по фігурах

Коли інструмент активний:

* при наведенні на фігуру показуються всі її anchor-и
* активний anchor підсвічується

Це в тебе вже частково є.

---

## Етап 3. MouseDown на anchor

Користувач натискає на anchor фігури.

У цей момент **ще не створюється повноцінний connector у списку edges**, а запускається draft/pending стан.

Наприклад:

```js
draftConnector: {
  start: {
    attached: { type: "node", nodeId, anchor },
    x,
    y
  },
  currentPointer: { x, y },
  template: {
    pathType,
    endcaps,
    behavior
  }
}
```

---

## Етап 4. Drag preview

Поки користувач тягне:

* кінець draft connector-а слідує за мишкою
* якщо наведено на інший anchor — preview прилипає до нього
* geometry preview перебудовується на льоту

Тобто на canvas показується **тимчасова лінія**, але вона ще не збережена як справжній connector.

---

## Етап 5. MouseUp

### Якщо відпустив на anchor

створюється реальний connector:

* `start` attached до першої фігури
* `end` attached до другої фігури
* `pathType/endcaps/behavior` беруться з шаблону

### Якщо відпустив не на фігурі

тут є 2 варіанти:

#### варіант A

не створювати нічого

#### варіант B

створити connector з:

* `start` attached до фігури
* `end` detached у точці відпускання

Для твоєї задачі я б радив **варіант B**, бо він ближчий до draw.io і зручніший.

---

# Головна архітектурна порада

Тобі потрібна окрема сутність:

## `draftConnector`

або

## `pendingConnector`

Це дуже важливо.

Не треба змішувати:

* редагування вже існуючого connector-а
* і процес створення нового

# Яку модель я б радив

У store додати:

```js
connectorCreation: {
  isActive: false,
  template: null, // { pathType, endcaps, behavior }
  draft: null     // тимчасовий connector під час drag
}
```

Або простіше:

```js
connectorTool: {
  activeTemplate: null
}

draftConnector: null
```

# Найчистіший варіант

## 1. `activeConnectorTool`

Описує, який тип зараз будується

```js
{
  pathType: "orthogonal-elbow",
  endcaps: { start: "none", end: "none" },
  behavior: { type: "default", link: null }
}
```

## 2. `draftConnector`

Описує поточний drag-процес

```js
{
  start: {...},
  end: {...},
  pathType: "...",
  endcaps: {...},
  behavior: {...},
  geometry: { points: [...] }
}
```

---

# Які події тобі треба додати

---

## 1. `activateConnectorTool(template)`

Кнопка в панелі не створює connector, а лише активує інструмент.

Наприклад:

* ламана лінія
* стрілка
* пряма

---

## 2. `startConnectorFromAnchor(anchor)`

Викликається при натисканні на anchor фігури, якщо активний connector tool.

Створює `draftConnector`.

---

## 3. `updateDraftConnector(pointer)`

Під час move:

* оновлює кінець draft connector-а
* якщо є attach preview — прилипає до нього
* перебудовує geometry

---

## 4. `finishDraftConnector(targetAnchor | pointer)`

Створює вже реальний connector і кладе в `edges`.

---

## 5. `cancelDraftConnector()`

Escape / кнопка відміни / клік у порожнє місце

---

# Як це вбудувати в твою поточну систему

У тебе вже є:

* `createConnector`
* `rebuildConnectorGeometry`
* `getConnectorAttachPreview`
* `handleConnectorStartDragMove / End`
* `handleConnectorEndDragMove / End`

Це добре, бо новий принцип побудови не ламає стару систему.
Тобі просто треба додати **ще один режим використання**.

# Правильне розділення режимів

## Режим A — створення нового connector-а

Працює через:

* `activeConnectorTool`
* `draftConnector`

## Режим B — редагування існуючого connector-а

Працює як зараз:

* можна від’єднати start/end
* переприв’язати
* редагувати точки

Це ідеально стикується.

---

# Що треба зробити в UI

Я б радив таку поведінку кнопок:

## Кнопка “Ламана”

Не вставляє лінію на canvas одразу, а:

* активує tool
* курсор змінюється
* користувач починає створення drag-ом від фігури

## Кнопка “Стрілка”

Так само:

* активує tool
* далі побудова від фігури до фігури

## Кнопка “Скасувати”

Скидає:

* `activeConnectorTool`
* `draftConnector`

Це важливо для UX.

# Як виглядає правильний flow

## Варіант 1 — object to object

1. активував інструмент
2. натиснув на anchor першої фігури
3. потягнув
4. навів на anchor другої
5. відпустив
6. connector створено

## Варіант 2 — object to free point

1. активував інструмент
2. натиснув на anchor фігури
3. потягнув
4. відпустив у canvas
5. connector створено, але другий кінець detached

## Варіант 3 — потім reattach

1. потягнув detached endpoint
2. прив’язав до іншої фігури

У тебе це вже майже є.

# Який preview треба показувати

Під час побудови треба рендерити не збережений edge, а окремий draft layer.

Тобто:

* `ConnectorsLayer` — рендерить існуючі edges
* `DraftConnectorLayer` — рендерить тільки тимчасовий connector

Це дуже чиста архітектура.

---

# Чому не треба одразу класти draft в edges

Бо тоді під час drag у тебе буде:

* “напівстворений” connector в основному списку
* складно відрізнити його від завершених
* складніше відміняти створення
* складніше керувати selection/context menu

Тому `draftConnector` окремо — значно краще.

---

# Як це стикується з draw.io принципом

Саме так draw.io і сприймається користувачем:

* інструмент активний
* ти починаєш лінію від об’єкта
* поки тягнеш — це preview
* після drop — реальний об’єкт

# Які поля треба в draftConnector

Я б радив майже ту ж модель, що й у реального connector-а:

```js
{
  kind: "connector",
  pathType,
  start,
  end,
  route: {
    baseOffset: null,
    waypoints: []
  },
  endcaps,
  behavior,
  geometry: {
    points: []
  },
  style
}
```

Тобто draft — це майже той самий connector, тільки не збережений у `edges`.

Це дуже зручно, бо:

* той самий builder geometry
* той самий preview rendering
* мінімум дублювання логіки

---

# Найважливіший архітектурний висновок

Тобі не треба міняти поточну модель connector-а.
Тобі треба додати над нею **режим створення**:

* `activeConnectorTool`
* `draftConnector`

І тоді ти отримаєш саме draw.io-подібний принцип побудови.

---

# Коротко

Щоб ламана лінія будувалась як у draw.io, а не вставлялась у фіксоване місце, треба:

* кнопка активує **інструмент**, а не одразу створює edge
* від фігури запускається **draft connector**
* під час drag рендериться preview
* на drop створюється справжній connector
* далі його можна від’єднувати й переприв’язувати так, як у тебе вже зроблено

---

Якщо хочеш, наступним повідомленням я можу дати тобі вже **готовий MVP план по файлах**:

* що додати в store
* який новий `DraftConnectorLayer`
* які методи в controller
* і де саме в `NodesLayer/anchor click` запускати створення нового connector-а.
